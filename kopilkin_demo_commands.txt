9. REDIS RATE LIMITING DEMO
============================================================

Normal health check:

curl http://127.0.0.1:8088/health

To demonstrate rate limiting clearly:
1. Temporarily set RATE_LIMIT_PER_MINUTE to "5" in docker-compose.yml for api-gateway-1 and api-gateway-2.
2. Restart gateways and balancer:

docker compose up -d --build api-gateway-1 api-gateway-2 nginx-balancer

3. Run several requests quickly:

for i in {1..10}; do curl -s http://127.0.0.1:8088/health; echo; done

Expected after limit is exceeded:
{"detail":"Rate limit exceeded. Please try again later.","limit_per_minute":5}

After demo, set RATE_LIMIT_PER_MINUTE back to "60" and restart:

docker compose up -d --build api-gateway-1 api-gateway-2 nginx-balancer

============================================================
17. SHORT DEFENSE SCRIPT
============================================================

What we implemented:

1. PostgreSQL persistence:
   auth-service owns auth_db.
   transaction-service owns transaction_db.
   savings-service owns savings_db.

2. Kafka / EDA:
   Services publish domain events after business operations:
   user.registered, transaction.created, transaction.deleted,
   goal.created, goal.updated, recommendation.requested, recommendation.generated.

3. Redis:
   Redis is used for transaction summary cache, user profile cache, and rate limiting counters.

4. Routing layer:
   Frontend sends API requests to Nginx on port 8088.
   Nginx balances traffic between api-gateway-1 and api-gateway-2.
   API Gateway routes requests to internal services and applies Redis-based rate limiting.

5. RecSys:
   recsys-service generates smart recommendations using:
   heuristic rules,
   content-based recommendation,
   educational collaborative filtering with synthetic user profiles.

6. AI:
   agent-service uses local Ollama, Qdrant vector memory and Langfuse tracing.

7. Infrastructure:
   Full local infrastructure runs through Docker Compose.
   Kubernetes/Talos/Cilium/ArgoCD/Helm are described as future production direction in the report.
